What H3 Is

H3 is not another agent framework. It is a protocol — a clean contract between platform and brain.

🔌 Protocol, Not Platform

Two endpoints. REST or gRPC. Any language. Zero Hermes code changes to add a new harness. The protocol is the integration surface.

🧠 Harness Owns Thinking

Harnesses decide what to do. Hermes executes it. The harness never touches files, keys, or gateways. It thinks. Hermes does.

✅ Compliance-Gated

A 43-test battery runs against any harness endpoint. Exit code 0 = H3-compliant. No harness code changes needed to be tested.

🌐 Polyglot SDKs

First-class SDKs for Go, Python, and TypeScript. Generated types from the OpenAPI 3.1 spec. Mock Hermes test beds ship with each language.

🔐 Secure by Design

Harnesses run locally on the Hermes host. They cannot access credentials directly or bypass tool sandboxing. Hermes authenticates users; the harness trusts Hermes.

🔄 Graceful Degradation

Harness unreachable? Sessions fall back to Hermes native agent loop automatically. Health-polling re-adds harnesses to the pool when they recover.

The Decision Protocol

Every harness response is exactly one Decision. Six types cover every possible interaction pattern.

tool_call

Execute a Tool

Harness requests Hermes to run a tool (terminal, web search, file ops, etc.). Hermes executes in a sandbox and returns the result.

llm_call

Run an LLM Prompt

Harness delegates a sub-prompt to a specific model. Hermes handles provider routing, rate limits, and returns the response.

text

Send Text to User

Harness sends markdown-formatted text to the user. Can be a partial message (finished: false) or the final response (finished: true).

wait

Pause for External Signal

Harness tells Hermes to wait — for CI to finish, for a human to respond, for an external event. Hermes polls and resumes automatically.

delegate

Spawn a Sub-Agent

Harness offloads a task to a sub-agent. Hermes spawns it, collects the result, and feeds it back through the loop.

end

Terminate Session

Harness signals completion — task finished, user cancelled, error, timeout, or rate-limited. Session resources are released.

Security Model

🏠 Local-Only Binding

Harnesses bind to localhost. No network exposure by default. Hermes communicates over the loopback interface.

🔑 Credential Isolation

Harnesses CANNOT access Hermes credential vault directly. All API keys, tokens, and secrets stay inside the platform.

📦 Tool Sandboxing

All tool execution is sandboxed and attributed to the harness session. Harnesses cannot bypass sandboxing or escalate privileges.

👤 Identity Forwarding

Hermes authenticates the user. The harness receives context.identity and decides authorization. The harness trusts Hermes.

📋 Audit Trail

Every tool execution, LLM call, and decision is logged with the harness session ID for full traceability.

🔄 Session Routing

Per-session, per-platform, or per-user harness routing. Unreachable harnesses silently fall back to the native agent loop.

Error Modes & Recovery

Every failure mode has a defined behavior. No silent failures.

FailureBehavior
Harness unreachable at startupLog warning, route to native agent loop
Harness timeout mid-sessionRetry 3× with exponential backoff (1s, 2s, 4s)
Harness returns malformed DecisionLog error, return error to user, end session
Harness hangs (no response)Timeout at configured threshold, terminate session
Harness returns end with error reasonRelay error to user, terminate session
Health check fails 3× consecutivelyRemove from pool, re-add on first successful check
