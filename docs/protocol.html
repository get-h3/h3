H3 Protocol Reference

A narrow two-endpoint REST contract between Hermes and any agent harness. One JSON wire format across Go, Python, and TypeScript.

01 / API Surface

Endpoints

All endpoints live under http://localhost:9191/v1/. The harness exposes them; Hermes calls them.

Method Path Description Request / Response
GET /v1/health Health check Hermes polls this every 30 seconds. Three consecutive failures mark the harness unreachable and sessions route to native fallback.
→ 200: HealthResponse (status: "ok")
→ 503: HealthResponse (status: "degraded"/"down")
POST /v1/process New message → Decision Hermes calls this when a new user message arrives. Harness receives full context (history, tools, models, memory, config) and must return exactly one Decision.
← Body: ProcessRequest
→ 200: Decision
→ 400 | 500 | 504
POST /v1/result Execution result → Decision Hermes calls this after executing a prior Decision. Loop continues until the harness returns decision type "end".
← Body: ResultRequest
→ 200: Decision
→ 400 | 404 | 410 | 500
POST /v1/cancel Cancel in-flight Hermes calls this to cancel an in-flight decision. Used when user sends a new message mid-processing or types /stop. Harness should abort current operation ASAP.
← Body: CancelRequest
→ 200: {cancelled, cancelled_decision_id}
→ 404 | 500
GET /v1/sessions/{session_id} Session metadata Returns metadata for an active or completed session.
→ 200: SessionResponse
→ 404 | 500
DELETE /v1/sessions/{session_id} Terminate session Forces session termination. Used for cleanup and /new command handling. Harness should release all resources for this session.
→ 200: {terminated, session_id}
→ 404 | 500

02 / Decision Types

Decisions

Every /v1/process or /v1/result call must return exactly one Decision. The decision field is a discriminator — six possible values.

⚙

tool_call

Execute a Hermes tool. Provide the tool name and JSON Schema-conforming params. Hermes runs the tool and calls /v1/result with the output.

🧠

llm_call

Run an LLM prompt through Hermes. Specify model, messages, temperature, and optional tools. Hermes handles tokenisation and returns the response.

💬

text

Send text to the user. The message appears in their chat. Mark finished: true to show the response as complete; false for streaming indicators.

⏳

wait

Pause for an external signal. Hermes hangs until the timeout or an external trigger arrives, then calls /v1/result.

🔀

delegate

Spawn a sub-agent. Provide a goal, role, and optional context. Hermes runs the sub-agent in a separate session and returns the result.

🏁

end

Terminate the session. Provide a reason (task_complete, user_requested, error, timeout, rate_limited, cancelled) and optional summary.

Example: tool_call Decision

{
  "decision": "tool_call",
  "decision_id": "d_a1b2c3d4",
  "tool_call": {
    "name": "web_search",
    "params": {
      "query": "H3 brain-swap protocol",
      "limit": 5
    },
    "reasoning": "Search for H3 documentation to answer user question"
  }
}

Example: end Decision

{
  "decision": "end",
  "decision_id": "d_z9y8x7w6",
  "end": {
    "reason": "task_complete",
    "summary": "Found 3 results for H3 protocol. Delivered to user."
  }
}

03 / Wire Format

JSON Schema Examples

All 14 schemas live under schemas/v1/. These are the core types you'll work with in every harness.

Decision — Discriminated union

The decision field determines which sub-schema applies. Required fields: decision, decision_id.

{
  "$id": "schemas/v1/decision.json",
  "title": "Decision",
  "type": "object",
  "required": ["decision", "decision_id"],
  "properties": {
    "decision": {
      "type": "string",
      "enum": ["tool_call", "llm_call", "text", "wait", "delegate", "end"]
    },
    "decision_id": { "type": "string" }
  },
  "oneOf": [
    { "description": "tool_call", "properties": {"decision":{"const":"tool_call"},"tool_call":{"$ref":"tool-call.json"}}, "required":["tool_call"] },
    { "description": "llm_call",  "properties": {"decision":{"const":"llm_call"},"llm_call":{"$ref":"llm-call.json"}},    "required":["llm_call"] },
    { "description": "text",      "properties": {"decision":{"const":"text"},"text":{"$ref":"text-response.json"}},        "required":["text"] },
    { "description": "wait",      "properties": {"decision":{"const":"wait"},"wait":{"$ref":"wait.json"}},                  "required":["wait"] },
    { "description": "delegate",  "properties": {"decision":{"const":"delegate"},"delegate":{"$ref":"delegate.json"}},      "required":["delegate"] },
    { "description": "end",       "properties": {"decision":{"const":"end"},"end":{"$ref":"end.json"}},                     "required":["end"] }
  ]
}

ToolCall — Execute a Hermes tool

Required fields: name, params. Optional: reasoning (audit trail).

{
  "$id": "schemas/v1/tool-call.json",
  "title": "ToolCall",
  "type": "object",
  "required": ["name", "params"],
  "properties": {
    "name": { "type": "string", "description": "Tool name matching one from context.tools." },
    "params": { "type": "object", "description": "Parameters matching the tool's JSON Schema." },
    "reasoning": { "type": "string", "description": "Why this tool call — logged for auditing." }
  }

ProcessRequest — POST /v1/process body

Hermes sends the full world state: session, message, identity, and context (history, tools, models, config, memory). Required: session_id, message, identity, context.

{
  "$id": "schemas/v1/process-request.json",
  "title": "ProcessRequest",
  "type": "object",
  "required": ["session_id", "message", "identity", "context"],
  "properties": {
    "session_id": { "type": "string" },
    "message":    { "$ref": "common.json#/definitions/Message" },
    "identity":   { "$ref": "common.json#/definitions/Identity" },
    "context":    { "$ref": "common.json#/definitions/Context" }
  }
}
Context structure (common.json) 
Context {
  history: HistoryEntry[]         // Last N messages
  tools: Tool[]                   // Available Hermes tools
  models: Model[]                 // Available LLMs with pricing
  memory?: string                 // DuckBrain memory
  skills?: string[]               // Loaded skill names
  config: Config                  // max_iterations, timeout_seconds, project_dir, etc.
  session_state: SessionState     // turn_count, cost_so_far, started_at, etc.
}

Tool {
  name: string
  description: string
  parameters: object              // JSON Schema for tool parameters
}

Model {
  name: string
  provider: string
  context_window: integer
  supports_vision?: boolean
  supports_tool_calling?: boolean
  cost_per_1k_input?: number
  cost_per_1k_output?: number
}

HealthResponse — GET /v1/health response

Required: status ("ok"|"degraded"|"down"), version. Optional: uptime_seconds, active_sessions, capabilities, degraded_reason, error.

{
  "$id": "schemas/v1/health-response.json",
  "title": "HealthResponse",
  "type": "object",
  "required": ["status", "version"],
  "properties": {
    "status":           { "type": "string", "enum": ["ok", "degraded", "down"] },
    "version":          { "type": "string" },
    "transport":        { "type": "string", "enum": ["rest"] },
    "protocol_version": { "type": "string" },
    "uptime_seconds":   { "type": "integer", "minimum": 0 },
    "active_sessions":  { "type": "integer", "minimum": 0 },
    "capabilities":     { "type": "array", "items": {"type":"string","enum":["tool_call","llm_call","text","wait","delegate","end"]} },
    "degraded_reason":  { "type": "string" },
    "error":            { "type": "string" }
  }
}

04 / Errors

Error Codes

All error responses follow the ErrorResponse schema: { error: { code, message, details? } }.

HTTP Code Description
400 INVALID_REQUEST Malformed JSON or missing required fields
400 INVALID_DECISION Decision type not recognized
400 UNKNOWN_TOOL Tool name not in available tools
400 UNKNOWN_MODEL Model not in available models
404 SESSION_NOT_FOUND Session ID doesn't exist
410 SESSION_EXPIRED Session timed out
504 HARNESS_TIMEOUT Harness didn't respond in time
500 INTERNAL_ERROR Harness-side crash

Example: 404 SESSION_NOT_FOUND

HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": {
    "code": "SESSION_NOT_FOUND",
    "message": "Session s_abc123 does not exist on this harness."
  }
}

05 / Loop

Protocol Flow

The core loop: Hermes sends a message, your harness returns a decision, Hermes executes it, sends the result back. Repeat until "end."

1. User sends messageHermes calls POST /v1/process with full context
2. Loop: execute → resultHermes executes decision, calls /v1/result
3. End or cancelHarness returns "end" or Hermes cancels
