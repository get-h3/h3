Build Your First H3 Harness

A step-by-step guide to creating an H3-compliant agent harness from scratch. Ship the brain. Keep the platform.

Before you start

What you'll need

This guide walks through building a bare-minimum harness that responds to the two core H3 callbacks and passes the compliance test battery.

🔧One language runtime — Go 1.22+, Python 3.10+, or Node.js 18+ / Bun
📦A terminal — you'll install an SDK, run a server, and execute the test battery
🔌An open port — your harness listens on localhost:9191 (configurable)
⚡10 minutes — from zero to verified compliance

Step 2

Implement the harness interface

The H3 protocol defines two callbacks: onProcess (when Hermes sends a message) and onResult (when Hermes reports back). Implement both to close the loop.

Minimal echo harness

This echo harness repeats every message back to Hermes, then closes the session. It's the simplest valid H3 harness and passes the full test battery.

// main.go — minimal H3 echo harness (Go)
package main

import (
  "fmt"
  "net/http"
  "github.com/get-h3/sdk-go/harness"
  "github.com/get-h3/sdk-go/protocol"
)

type Echo struct{}
func (Echo) OnProcess(r *protocol.ProcessRequest) (*protocol.Decision, error) {
  return &protocol.Decision{Decision: protocol.DecisionText,
    DecisionID: "echo-1", Text: &protocol.TextResp{Content: fmt.Sprintf("Echo: %s", r.Message.Content), Finished: true}}, nil
}
func (Echo) OnResult(*protocol.ResultRequest) (*protocol.Decision, error) {
  return &protocol.Decision{Decision: protocol.DecisionEnd, DecisionID: "done", End: &protocol.End{Reason: protocol.EndTaskComplete}}, nil
}
func (Echo) OnCancel(*protocol.CancelRequest) error { return nil }
func (Echo) OnSessionTerminate(string) error { return nil }
func (Echo) Health() *protocol.HealthResponse {
  return &protocol.HealthResponse{Status: protocol.HealthOK, Version: "1.0.0", Transport: "rest", ProtocolVersion: "1.0", Capabilities: []protocol.DecisionType{protocol.DecisionText, protocol.DecisionEnd}}
}
func main() { fmt.Println("H3 harness listening on :9191"); http.ListenAndServe(":9191", harness.NewHTTPServer(Echo{})) }
# main.py — minimal H3 echo harness (Python)
from fastapi import FastAPI
from h3_harness import BaseHarness, Decision, DecisionType, End, TextResponse, create_router

class Echo(BaseHarness):
    async def on_process(self, req):
        return Decision(decision=DecisionType.TEXT, text=TextResponse(content=f"Echo: {req.message.content}", finished=True))

    async def on_result(self, req):
        return Decision(decision=DecisionType.END, end=End(reason="task_complete"))

app = FastAPI()
app.include_router(create_router(Echo()))
// index.ts — minimal H3 echo harness (TypeScript)
import { Hono } from "hono";
import { createH3Router } from "@get-h3/h3-harness-sdk";
import type { Harness, Decision } from "@get-h3/h3-harness-sdk";

const echo: Harness = {
  async onProcess(req) { return { decision: "text", decision_id: crypto.randomUUID(), text: { content: `Echo: ${req.message.content}`, finished: true } }; },
  async onResult(_req) { return { decision: "end", decision_id: crypto.randomUUID(), end: { reason: "task_complete" } }; },
  health() { return { status: "ok", version: "1.0.0", transport: "rest", protocol_version: "1.0", capabilities: ["text", "end"] }; }
};
const app = new Hono(); app.route("/", createH3Router(echo)); export default app;

What's happening

Every H3 harness must satisfy five methods:

  • Health() — reports status, version, and supported decision types to the shim
  • OnProcess() — receives the user's message and returns a decision (text, tool call, wait, etc.)
  • OnResult() — receives Hermes' execution result so you can decide the next step
  • OnCancel() — called when the user or system cancels the session
  • OnSessionTerminate() — called when Hermes ends the session

The echo harness above returns a text decision on every process call and an end decision on the first result. That's enough to complete a full H3 conversation loop.

Start your harness

Quick health check

In another terminal, hit the health endpoint to confirm your harness is alive:

curl http://localhost:9191/health

Expected response: {"status":"ok","version":"1.0.0","transport":"rest","protocol_version":"1.0","capabilities":["text","end"]}

Step 4

Run the compliance test battery

The H3 test battery exercises 43 scenarios across 6 functional regions. Point it at your harness to verify protocol compliance.

pip install hermes-h3-shim
h3-test --endpoint http://localhost:9191

The test battery runs in under 5 seconds. You'll see each test pass/fail with a final score out of 43.

Scoring 43/43? Your harness is fully H3-compliant. Add the badge to your README and you're ready to swap brains. See the compliance section for badge URLs.

Step 5

Wire it into Hermes

Tell the shim where your harness lives. From now on, Hermes will route process and result calls through the H3 boundary.

hermes h3 install
hermes h3 scaffold --lang go --output my-h3-harness
hermes h3 verify

hermes h3 install registers the shim as a Hermes plugin. hermes h3 scaffold generates a harness template (with SDK wiring already set up). hermes h3 verify confirms the plugin registration and harness reachability.
